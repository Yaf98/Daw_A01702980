¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?[1]
  • Validar los inputs es importante para:
    • Prevenir ciber ataques. Los ataques pueden ocacasionar que la página falle o incluso puede relevar información al atacante
    • Las aplicaciones no funcionen como lo esperamos. Si los datos no son evíados de manera correcta, seguramente tendremos
    • Si aseguramos que el usuario tienen una contraseña fuerte, a la vez estamos protegiendo sus datis

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?[2]
  • Las validaciones se pueden evadir desactivando JavaScript en el navegador que estemos usando
  • Otra opción es llenar el formulario e interceptar la información con algún software para que no llegue como se ingreso al servidor

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?
  • Para asegurar que los usuarios tengan contraseñas más seguras, protegiendo así sus datos
  • Para que nuestras páginas web funcionen como se espera que funcionen, si no validamos es problema que se presenten errores al llegar datos que no deberían haber llegado
